CAPEX Variance Commentary — Board Ready

1 of 25

Key Question

Is your CAPEX variance a cost problem or a timing problem? Do you know the difference before the board asks?

Act as a CAPEX Controller preparing a board report. Based on the variance data below [paste: project label, approved budget, actual spend, forecast-to-complete, variance — no project names or supplier details], write a concise commentary for each project.

Structure: headline variance → primary driver → timing vs cost distinction → risk flag if applicable.

Max 60 words per project.

🔒 Safe Input Guide

Replace project names with "Project Alpha / Beta / Gamma". Remove supplier names, contract values, and procurement-sensitive information.

capex-variance-analysis.prompt
$ YTD CAPEX Variance Analysis — vs Target / Outlook
Role You are a senior CAPEX Controller preparing a monthly variance review for senior management and the CFO.

Context Below is the YTD CAPEX spend data, anonymised, broken down by [product line / site / project]. Figures show actual spend vs Target and vs Latest Outlook in both absolute ($) and percentage terms.

Task — provide:
Top 5 YTD variances vs Target ranked by absolute $ value — flag adverse or favourable
Top 3 YTD variances vs Latest Outlook — identify re-phasing or scope changes
Root cause hypotheses for the largest variances
Recommended follow-up questions for project managers
CFO-ready executive summary — 3 sentences maximum
🔒 Safe Input Guide
Anonymise project names and cost centre codes
Replace absolute $ figures with % variances if sensitive
Remove site names, vendor names, and PII
Check your company's AI acceptable use policy first
